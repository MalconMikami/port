# Samples

Three example sites demonstrating Port's capabilities.

## Site A — Static Landing Page

`sample/site-a/` — An institutional landing page generated by AI. Pure static site, no backend.

```html
<!-- index.html — site-a -->
<section class="hero">
  <h1>Alpha Tech</h1>
  <p>We turn ideas into intelligent digital solutions.</p>
</section>
```

**Deploy:**
```bash
cd cli && npm install
npx tsx src/index.ts deploy ../sample/site-a
```

Access: [http://site-a.localhost:10000](http://site-a.localhost:10000)

**Demonstrates:**
- Static file serving with custom HTML + CSS
- CLI deploy with no configuration
- Subdomain routing (`site-a.*`)

## Sample CRUD — Full-stack Site with Backend

`sample/sample-crud/` — Complete CRUD application with frontend + server-side functions + database.

### Structure

```
sample-crud/
├── index.html                    ← Frontend with form and listing
├── css/style.css                 ← Custom styles
├── js/app.js                     ← Frontend logic with SDK
├── functions/
│   └── items.js                  ← Server-side RPC functions
└── img/                          ← Assets
```

### Frontend (`js/app.js`)

Uses the Port SDK for all backend communication:

```js
// Load user identity
const user = await window.port.user.get();

// Apply dynamic theme from config_public
const theme = await window.port.config.get('theme');
document.documentElement.style.setProperty('--primary', theme.primaryColor);

// CRUD via RPC — functions run on the server
await window.port.functions.items.create({ name: 'Keyboard', price: 299 });
const { docs } = await window.port.functions.items.list();
await window.port.functions.items.remove({ id: itemId });
```

### Server-Side Functions (`functions/items.js`)

Validation and business logic on the server, with database access:

```js
// Server-side validation
export async function create({ name, price }, { port, userId }) {
  if (!name || name.length < 2) throw new Error('Name must have at least 2 characters');
  if (price === undefined || price < 0) throw new Error('Invalid price');

  // Create in the site's database
  return port.db.collection('items').create({ name, price, owner: userId });
}

// List with optional filter
export async function list({ owner }, { port }) {
  const result = await port.db.collection('items').list();
  if (owner) result.docs = result.docs.filter(d => d.owner === owner);
  return result;
}

// Deletion with ownership check
export async function remove({ id }, { port, userId }) {
  if (!id) throw new Error('ID is required');
  const result = await port.db.collection('items').list();
  const item = result.docs.find(d => d.id === id);
  if (!item) throw new Error('Item not found');
  if (item.owner !== userId) throw new Error('Only the creator can delete this item');
  await port.db.collection('items').delete(id);
  return { ok: true };
}
```

### Deploy

```bash
cd cli && npm install
npx tsx src/index.ts deploy ../sample/sample-crud
```

Access: [http://sample-crud.localhost:10000](http://sample-crud.localhost:10000)

**Demonstrates:**
- Full `window.port` SDK (db, functions, config, user)
- RPC with server-side functions
- Validation and business logic on the backend
- Dynamic theme via config_public
- Complete CRUD with PostgreSQL
- Ownership and authorization (only the creator can delete)

## Creating Your Own Site

```bash
# Create a folder with index.html
mkdir my-site
echo '<h1>Hello, Port!</h1>' > my-site/index.html

# Add server-side functions
mkdir my-site/functions
cat > my-site/functions/hello.js << 'EOF'
export async function ping({ name }, { port }) {
  return { message: `Hello, ${name}!`, timestamp: new Date().toISOString() };
}
EOF

# Deploy
cd cli && npx tsx src/index.ts deploy ../my-site
```

Your site is at `http://my-site.localhost:10000`.

```js
// In the frontend:
const result = await window.port.functions.hello.ping({ name: 'Maria' });
// → { message: 'Hello, Maria!', timestamp: '2026-06-11T...' }
```
